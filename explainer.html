1. The Body ID Trick

Every page in this site shares one CSS file. To give each page its own unique background image, each <body> tag has a unique id. Then in CSS, you can target elements on a specific page by starting your selector with that id. 

<!-- index.html -->
<body id="home">

<!-- page1.html -->
<body id="milford">
/* Only applies on the home page */
#home header {
    background-image: url(images/home-bg.jpg);
}

/* Only applies on the Milford Sound page */
#milford header {
    background-image: url(images/milford-bg.jpg);
}

2. Google Fonts

Instead of using a font already installed on the user's computer, you can load any font from Google's library by pasting a <link> tag into your <head>. Then reference it in CSS just like any other font. 

<!-- In the <head> of your HTML -->
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@400;700&display=swap" rel="stylesheet">
body {
    font-family: 'Playfair Display', serif;
}

3. Background Images

Any element can have a background image set in CSS. The three key properties are background-size: cover (scales the image to fill the element completely, cropping if needed), background-repeat: no-repeat (prevents tiling), and background-position (controls which part of the image stays in frame). 

header {
    background-image: url(images/home-bg.jpg);
    background-size: cover;
    background-repeat: no-repeat;
    background-position: center 60%;
}

4. CSS Animations

CSS animations have two parts: a @keyframes block that defines what changes, and an animation property on the element that runs it. The forwards keyword means the element stays at the final keyframe value instead of snapping back. The 0.5s delay means the animation waits half a second before starting. 

@keyframes shrink-header {
    0%   { height: 100vh; }
    100% { height: 20vh; }
}

header {
    animation: shrink-header 1s ease 0.5s forwards;
    height: 100vh;
}

5. CSS Transitions

A transition smoothly animates a CSS property whenever it changes — usually on hover. You add transition to the element in its normal state, not in the :hover rule. That way it smoothly animates both entering and leaving the hover. 

/* Normal state — defines the transition */
nav > ul > li > a {
    transition: transform 0.3s ease;
}

/* Hover state — just sets the target value */
nav > ul > li:hover a {
    transform: scale(1.2);
}

6. Image Zoom on Hover

Zooming an image on hover uses two elements working together. The container gets overflow: hidden so the zoomed image is clipped to its original edges. The image inside uses transform: scale() on hover, paired with a transition to animate it smoothly. 

.home-pic-container {
    overflow: hidden;  /* clips anything outside the box */
    border-radius: 25px;
}

.home-pic-container > img {
    width: 100%;
    transition: transform 0.5s ease;
}

.home-pic-container:hover img {
    transform: scale(1.3);
}

7. The Nav Hover Photo Effect

Each nav item reveals a background image when hovered. Because background-image can't be transitioned directly, the image snaps in instantly — but the text already has a transition on its scale, so the combined effect still feels polished. 

.milford-item:hover {
    background-image: url(images/milford-icon.jpg);
    background-size: cover;
    background-position: center 80%;
}

nav > ul > li:hover a {
    color: white;
    text-shadow: 0px 0px 4px black, 0px 0px 8px black;
    transform: scale(1.2);
}

8. The ::after Pseudo-Element

A pseudo-element like ::after inserts a virtual element after an element's content — no extra HTML needed. Here it draws a growing underline beneath the active nav link. It starts at width: 0 and animates to width: 100%. Using margin: 0 auto makes it grow outward from the center rather than from the left. 

#active > a::after {
    content: '';           /* required, even if empty */
    display: block;
    height: 4px;
    background-color: black;
    width: 0;
    margin: 0 auto;        /* grow from center outward */
    animation: grow-underline 0.4s ease 1.5s forwards;
}

@keyframes grow-underline {
    from { width: 0; }
    to   { width: 100%; }
}

9. Flexbox

Flexbox is used throughout this site for layout. Setting display: flex on a container lets you control how its children are arranged. justify-content controls spacing along the main axis (horizontal by default), and align-items controls alignment on the cross axis (vertical by default). 

/* Centers the h1 inside the header */
header {
    display: flex;
    justify-content: center;
    align-items: center;
}

/* Puts the four nav items in a row, centered */
nav > ul {
    display: flex;
    justify-content: center;
}

10. Media Queries

A media query wraps CSS rules that only apply when the screen is a certain size. Everything inside the @media block overrides the normal rules when the condition is true. On this site, screens narrower than 768px get a stacked mobile layout. The order property is also used to ensure images always appear above text on mobile, regardless of their order in the HTML. 

@media (max-width: 768px) {

    nav > ul {
        flex-direction: column;
    }

    .home-layer {
        flex-direction: column;
    }

    /* Always put images above text, no matter the HTML order */
    .home-layer .home-pic-container {
        order: -1;
    }

}
